# 第一条 3DGS 实验链路

## 目标

用最小复杂度跑通一条从素材到可评估结果的空中版 `3DGS` 实验链路，为后续 `Web` 展示做决策。

## 推荐链路

第一轮推荐采用如下顺序：

1. 使用全量分层均匀抽样得到的 `PoC 001` 子集做素材输入
2. 使用 `COLMAP` 完成相机位姿恢复与稀疏重建
3. 使用 `3DGS` 训练方案产出 splat 结果
4. 对结果做基础裁切、清理与体积评估
5. 判断是否值得进入 `Web viewer` 集成阶段

## 为什么先这样做

- `COLMAP` + `3DGS` 是最常见也最容易对齐问题来源的基础链路
- 第一轮目的是验证素材可行性，不是马上追求最终展示效果
- 只有先跑通最基础链路，后面才能判断是否需要换训练工具、补素材或调整导出方案

## 第一轮实验最关心的问题

- 均匀抽样后的俯视五向图是否足以恢复稳定相机位姿
- 校园主结构是否可辨识
- 是否出现严重漂浮、拉花、噪点或缺面
- 产物体积是否大到完全不适合 `Web`
- 结果是否具备继续做纪念展示的价值

## 暂不做的事

第一轮先不做以下内容：

- 不做全量素材训练
- 不做前端集成
- 不做热点内容配置
- 不做移动端适配
- 不做登录、数据库和发布系统

## 实验产出要求

第一轮实验至少应产出：

- 一份实验记录，使用 `docs/asset-validation-template.md`
- 一版可截图或录屏的结果
- 对质量、体积、加载潜力的主观判断
- 对下一轮要扩大素材还是调整链路的结论

## 决策分支

如果结果较好：

- 扩大素材范围
- 尝试整批 `102MEDIA`
- 开始评估 `Web viewer` 集成可能性

如果结果一般：

- 调整抽样策略或人工补充分组
- 检查位姿恢复质量
- 排查素材覆盖与朝向混乱问题

如果结果很差：

- 先不要进入前端阶段
- 优先判断是否需要重新整理五向分类
- 评估是否需要补充素材或更换实验策略
