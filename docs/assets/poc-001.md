# PoC 001：首个素材子集选择

## 目标

为第一轮空中版 `3DGS` 可行性验证选择一个足够小、覆盖尽量均匀、便于快速暴露风险的素材子集。

## 修正说明

上一版曾将 `PoC 001` 设为一段连续序列，但这更接近“单一方向连续拍摄验证”，不适合当前“五向素材是否足以支撑重建”的判断目标。

因此本轮已改为：从全量素材中做分层均匀抽样，而不是取单段连续照片。

## 选择结论

首个 `PoC` 子集选择：基于 `assets/raw` 全量素材，按目录占比分层后做均匀抽样，共 `180` 张。

对应清单文件：`data/poc-001-files.txt`

当前分配结果：

- `assets/raw/101MEDIA`：抽取 `106` 张
- `assets/raw/102MEDIA`：抽取 `74` 张

## 选择依据

- 当前缺少可靠的角度标签，无法直接按“五向类别”精确分组
- 在这种情况下，基于全量素材做均匀抽样，比取单一连续序列更能覆盖不同拍摄方向与区域
- 分层抽样能保留两个素材目录的原始占比，减少某一目录过度代表整体的偏差
- 第一轮目标不是追求最终效果，而是尽快判断素材、训练链路与导出链路是否能跑通
- `180` 张的规模足以形成一个小型验证集，同时不会让训练和排查成本过高

## 当前假设

- 该子集比连续序列更适合验证“五向覆盖是否基本成立”
- 即便没有精确角度标签，均匀抽样也能更早暴露多方向素材混合后的问题
- 如果这一子集都无法得到可接受结果，就要优先怀疑素材覆盖、位姿恢复或实验链路，而不是前端展示

## 使用规则

- 所有脚本和记录都必须使用相对路径作为素材主标识
- 不使用文件名单独标识素材，因为存在跨目录重名且内容不同的图片
- 第一轮验证先只使用 `data/poc-001-files.txt` 中的图片
- 如需复现该清单，使用 `python3 scripts/select_poc_subset.py --sample-size 180`

## 通过标准

满足以下任一条件，可认为该 `PoC` 达到继续推进价值：

- 能完成位姿恢复并得到结构可辨识的场景
- 能训练出一版可查看的 splat，且校园主要结构没有完全崩坏
- 能导出到可进入 `Web` 集成评估的结果
