# PoC 001：首个素材子集选择

## 目标

为第一轮空中版 `3DGS` 可行性验证选择一个足够小、足够连续、便于快速暴露风险的素材子集。

## 选择结论

首个 `PoC` 子集选择：`assets/raw/102MEDIA/DJI_0001.JPG` 至 `assets/raw/102MEDIA/DJI_0180.JPG`。

对应清单文件：`data/poc-001-files.txt`

## 选择依据

- `102MEDIA` 总量较小，共 `676` 张，整体更适合做第一轮试验
- 该目录的修改时间集中，素材连续性更强，说明它更像单次或紧邻批次的拍摄结果
- 文件命名从 `DJI_0001.JPG` 连续开始，便于定义一个清晰、可复现的子集范围
- 第一轮目标不是追求最终效果，而是尽快判断素材、训练链路与导出链路是否能跑通
- `180` 张的规模足以形成一个小型验证集，同时不会让训练和排查成本过高

## 当前假设

- 该子集仍然属于俯视五向图范畴
- 该子集应能用于验证空中浏览视角下的基础可看性
- 如果这一子集都无法得到可接受结果，就要优先怀疑素材覆盖或实验链路，而不是前端展示

## 使用规则

- 所有脚本和记录都必须使用相对路径作为素材主标识
- 不使用文件名单独标识素材，因为存在跨目录重名且内容不同的图片
- 第一轮验证先只使用本清单中的图片，不混入其他目录素材

## 通过标准

满足以下任一条件，可认为该 `PoC` 达到继续推进价值：

- 能完成位姿恢复并得到结构可辨识的场景
- 能训练出一版可查看的 splat，且校园主要结构没有完全崩坏
- 能导出到可进入 `Web` 集成评估的结果

- 如果结果过差，再扩大到 `102MEDIA` 的更大连续区间或整批素材
