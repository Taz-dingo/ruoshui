# 项目 Spec

## 1. 项目名称

项目名：`若水广场`

名称取自学校旧日广场，兼具地点记忆与纪念意味。项目整体可理解为一个“老校区空中数字纪念馆”。

## 2. 背景

项目起源于对常州老校区的纪念。发起人在大学前两年曾在该校区生活，后来校区将被废弃或迁移，因此希望通过数字化方式保留其空间记忆。

这个项目的重点不是做一个功能复杂的平台，而是让人重新看到那个地方，并在浏览中重新感受到那段时间与空间。

## 3. 项目定位

- 产品定位：纪念向、回忆向、展示向的 `3D` 数字纪念项目
- 展示重点：空中视角的校园重访，而非地面步行式漫游
- 发布策略：优先 `Web`，保证尽可能低门槛访问
- 设备策略：第一版先只保证桌面端体验

## 4. 项目目标

### 4.1 核心目标

- 基于现有俯视五向航拍图，尝试重建老校区的空中版 `3DGS` 场景
- 在浏览器中提供可访问、可浏览、可导览的沉浸式纪念体验
- 通过少量策展式热点内容，增强“回到那里”的感受
- 用最小可行范围验证：素材是否足够、场景是否可看、页面是否可发布

### 4.2 第一版成功标准

- 能训练出一版可辨识的老校区空中 `3DGS` 结果
- 能在桌面浏览器中完成基本展示
- 用户能进入场景、切换机位、浏览校园主要区域
- 用户能查看少量代表性地点的图文记忆内容
- 整体体验不依赖安装原生客户端

### 4.3 非目标（第一阶段不做）

- 不优先做开放式用户发帖社区
- 不优先做登录、评论、审核等完整内容系统
- 不优先做小程序版本
- 不强行追求地面 walk-through 体验
- 不在第一阶段解决复杂后端与数据库设计

## 5. 用户与场景

### 5.1 目标用户

- 自己
- 曾在老校区生活过的同学与校友
- 对校园记忆、数字纪念项目感兴趣的人

### 5.2 核心使用场景

- 用户打开网页，进入老校区空中场景
- 用户通过镜头预设或简单操作浏览校园主要区域
- 用户在关键点位查看图文记忆内容
- 用户将某个固定视角或地点分享给他人

## 6. 产品范围

### 6.1 第一版必须有

- 一个可在 `Web` 打开的 `3DGS` 场景页
- 桌面端可用的浏览体验
- 场景加载状态与错误兜底
- 若干预设镜头或导览路径
- 若干策展式热点记忆点位
- 每个点位的图文内容展示层
- 一个简单的项目说明页

### 6.2 第一版可以有

- 背景音乐开关
- 镜头过渡动画
- 时间轴式地点导航
- 基础移动端兼容

### 6.3 后续版本再做

- 登录系统
- 内容后台或点位 CMS
- 用户投稿
- 视频内容投稿
- 小程序版本

## 7. 已知输入与关键假设

### 7.1 已知输入

- 现有航拍素材已经存放于 `assets/raw`
- 素材类型以俯视五向图为主
- 项目重点是空中展示，不做第一阶段的地面沉浸漫游

### 7.2 关键假设

- 现有俯视五向图足以支撑“空中版” `3DGS` 重建
- 对这个项目而言，视觉还原度优先于复杂交互
- `Web` 是最合适的第一发布端，因为它最容易被打开和分享
- 第一版无需后端也能完成核心体验

### 7.3 最大风险

- 素材视角不足，导致重建缺面、漂浮、噪声严重
- `3DGS` 原始资产过大，桌面浏览器加载过慢
- 空中视角可看，但热点内容不足以支撑纪念叙事
- `3DGS` Web viewer 与页面层的集成复杂度超预期

## 8. 技术路线（当前建议）

### 8.1 总体策略

- 第一版以 `Web` 为唯一发布端
- 第一版以 `3DGS` 为核心展示介质，不退回到 `mesh` 作为主方案
- 第一版尽量无后端，内容先静态化维护

### 8.2 资产生产链路

候选流程：

1. 清点与筛选 `assets/raw` 素材
2. 相机位姿恢复 / 稀疏重建
3. `3DGS` 训练
4. 对 splat 资产进行裁切、清理、压缩
5. 集成 `Web viewer`

### 8.3 页面与实现策略

- 页面框架：优先选择轻量、成熟、便于快速搭建的 `Web` 技术栈
- 场景层：优先集成成熟的 `3DGS Web Viewer`
- 外层页面：承载项目背景、导航、信息面板与热点内容
- 内容层：第一版先用本地 `JSON` 或静态内容维护
- 部署：优先静态站点或轻量 `Node` 部署

### 8.4 第一版内容数据结构

第一版尽量无后端，内容先静态化：

- `scene`
  - `id`
  - `title`
  - `description`
  - `assetUrl`
  - `previewImage`
- `cameraPreset`
  - `id`
  - `name`
  - `description`
  - `position`
  - `target`
- `memoryPoint`
  - `id`
  - `title`
  - `summary`
  - `position`
  - `cover`
  - `body`

## 9. 页面信息架构

### 9.1 首页

- 项目标题“若水广场”
- 老校区背景介绍
- 进入场景按钮
- 纪念说明与项目缘起

### 9.2 场景页

- `3DGS` 主视图
- 加载进度
- 镜头预设切换
- 热点入口
- 信息面板

### 9.3 点位内容面板

- 标题
- 地点说明
- 图文内容
- 返回场景

## 10. 开发原则

- 先验证最关键风险，再扩展功能
- 每次迭代只解决一个主要问题
- 文档先行，结果可验证
- 能静态化解决的问题，先不要引入后端
- 不为未来功能提前设计过多复杂结构
- 尽快做出第一版“可看、可判断”的结果
