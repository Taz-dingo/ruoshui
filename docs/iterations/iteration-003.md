# Iteration 003

## 目标

把下一轮 `GS` 模型优化从方向判断落成可执行实验计划，优先验证哪些动作最可能同时改善画面质量、几何稳定性与 `Web` 可交付性。

## 当前状态

本轮开始时：

- `Iteration 001` 已证明当前素材与训练链路可以得到主观上可接受的校园主结构
- `Iteration 002` 已证明当前导出链路的主要瓶颈是交付体积和底部大尺度离群高斯，而不是 viewer 本身
- 当前默认预览组合已收敛为 `crop-p05-p995 + 14x`
- 当前下一步不再是继续微调 viewer，而是从 `GS` 模型侧减少脏高斯、缩小有效范围，并判断是否值得切换到更适合几何和大场景的 `GS` 变体

## 本轮要产出的东西

- 一份按优先级排列的 `GS` 模型优化实验清单
- 每条实验路线的输入、验证方式、停止条件
- 一个明确的先后顺序，避免同时开太多不确定分支
- 一份算法路线筛选框架，避免默认把当前 `splatfacto` 当成唯一主线

## 核心判断

- 当前最先要验证的不是“换哪个 viewer”，而是“哪类 `GS` 表示最适合当前校园大场景”
- 当前最有把握的低成本动作应优先于高成本复训
- 只有当低成本动作不足以明显改善边缘噪声、有效范围或交付体积时，才进入更重的重处理与复训分支
- 现阶段仍不建议直接从 `180` 张跳到 `1600+` 张全量训练
- 算法路线判断至少要同时覆盖三类问题：几何精度、场景规模、交付体积

## 候选实验卡

### A1. 精细空间裁切

目标：

- 在现有 `rgb PLY` 与 viewer 基线上，继续验证比 `crop-p05-p995` 更贴近校园主体的空间裁切

输入：

- `outputs/iteration-002/export-rgb/splat.ply`
- `outputs/iteration-002/analysis/export-rgb-summary.json`
- `scripts/gaussian_ply_tools.py`

验证方式：

- 生成 `1-2` 组更激进但仍可接受的裁切候选
- 对比文件体积、bbox 变化、保留高斯比例和浏览器主观观感

停止条件：

- 如果体积范围明显缩小且主体观感不被破坏，则保留为后续预览或导出基线
- 如果裁切只能略微减小字节体积，且主体损伤明显，则不继续在纯后处理上深挖

### A2. 底部离群高斯定向清理

目标：

- 把已确认的 `z` 轴尾部大尺度离群高斯问题，进一步收敛成可重复的定向清理策略

输入：

- `outputs/iteration-002/export-rgb/splat.ply`
- `outputs/iteration-002/crop-zmin-p005/splat.ply`
- `scripts/gaussian_ply_tools.py`

验证方式：

- 针对 `z` 下界和极端尺度代理做小步参数扫描
- 记录每组参数下的保留比例、bbox 收缩和主观脏区改善情况

停止条件：

- 如果少量定向裁切即可显著改善底部脏区，则把这条路保留为默认清理前置步骤
- 如果必须牺牲大量主体高斯才有改善，则判定仅靠导出后清理不足以解决主问题

### B1. Per-image campus mask 方案设计

目标：

- 判断是否值得为下一轮 `ns-process-data` / 训练引入逐图校园遮罩，减少校外区域与边缘背景干扰

输入：

- `docs/iterations/iteration-001-validation.md`
- 当前 `processed` 数据格式与 `transforms.json` 的 `mask_path` 能力

验证方式：

- 明确遮罩格式、目录组织、最小标注范围和接入位置
- 先形成一版只覆盖小样本的操作方案，而不是一次性全量制作

停止条件：

- 如果遮罩制作成本可控，且与当前噪声分布高度对应，则进入小规模样本遮罩实验
- 如果问题主要来自视角混杂或姿态恢复，而不是校外边缘，则把遮罩降为次优先级

### B2. 素材重组与小规模复训

目标：

- 在不直接全量扩容的前提下，验证更结构化的输入组织是否能优先改善主观质量和交付结果

输入：

- `PoC 001` 当前 `180` 张子集
- 五向分组、连续段分组或 `300-600` 张级别的结构化扩容候选

验证方式：

- 先明确 `1-2` 个最小可执行重组方案
- 每个方案都要求同时记录输入规模、位姿恢复代价、训练时长预估和预期改善点

停止条件：

- 如果存在明显更干净的组织方式且成本可控，再进入新的 `COLMAP + splatfacto` 实验
- 如果只是“多加一点图看看”，但没有清晰结构假设，则不启动复训

### C1. `2DGS` / 几何优先路线筛选

目标：

- 判断 `2DGS` 一类方法是否更适合当前“校园主体表面 + 校外边缘干扰明显”的问题形态

输入：

- 当前 `splatfacto` 结果暴露出的 `floaters`、边缘拉花和底部大尺度离群高斯
- `2D Gaussian Splatting` 官方实现与文档
- `Mip-Splatting` 的抗锯齿与伪影抑制能力

验证方式：

- 先做文档级适配判断，而不是立刻迁移整条训练链路
- 重点比较 `2DGS` / `Mip-Splatting` 对当前项目的三个价值：几何更稳、边缘更干净、是否仍能承受训练和导出成本

停止条件：

- 如果 `2DGS` 路线主要提升的是几何和网格提取，但对 `Web` 体积与大场景训练成本帮助有限，则保留为次主线
- 如果 `2DGS` 路线同时能明显缓解当前脏边缘与大场景几何问题，则升级为下一轮实验候选

### C2. 大场景 `3DGS` 优化路线筛选

目标：

- 判断当前校园空中场景是否更适合采用带结构化表示或分块训练能力的大场景 `GS` 方案

输入：

- `Scaffold-GS`
- `Octree-GS`
- `CityGaussian / CityGaussianV2`
- `VastGaussian`

验证方式：

- 逐项判断这些路线分别解决什么问题：视角自适应、`LOD`、分块训练、并行训练、内存控制、大场景渲染
- 只保留与当前项目强相关的能力，不把“论文更强”直接等价为“项目更适合”

停止条件：

- 如果某条路线的核心收益主要落在超大城市场景或多卡并行，而当前校园规模还没到这个量级，则只做备选
- 如果某条路线能直接回应当前问题，例如 `LOD`、分块加载、内存和高斯数量控制，则提升优先级

### C3. 压缩与轻量化路线筛选

目标：

- 判断当前项目是否需要把“训练表示”和“交付表示”拆开，用压缩型 `GS` 方案作为交付端主线

输入：

- 当前 `rgb PLY` 约 `267 MiB`、压缩后仍约 `200 MiB`
- `LightGaussian` 一类压缩与量化路线

验证方式：

- 重点看它解决的是离线压缩、运行时渲染还是训练期高斯数量控制
- 判断其是否能与现有训练输出衔接，而不是要求整条链路完全重做

停止条件：

- 如果压缩路线主要依赖离线蒸馏和量化，且能与现有结果衔接，则保留为交付侧重点
- 如果它必须建立在另一套训练栈上，且当前实现成本过高，则暂列二级备选

## 优先级与执行顺序

1. 先做 `C1 / C2 / C3` 的文档级算法筛选，避免在错误训练主线上继续深挖
2. 再做 `A2`，因为底部大尺度离群高斯是当前最明确、最便宜、最可能立刻改善观感的问题
3. 再做 `A1`，把现有空间裁切从“分位数近似”推进到“更贴近校园主体”的候选基线
4. 然后做 `B1`，判断遮罩是否值得进入下一轮数据处理
5. 最后才进入 `B2`，只在前面证据足够时启动更贵的复训链路

## 本轮通过标准

满足以下条件即可认为本轮完成：

- 下一轮工作不再只是“继续优化看看”，而是有明确优先级的实验清单
- 每条候选路线都具备输入、验证方式和停止条件
- 团队知道下一步最小动作应先完成算法路线筛选，再决定是否继续沿当前 `splatfacto` 主线深挖

- 先补一份算法路线筛选记录，明确 `2DGS`、大场景 `3DGS` 与压缩型 `GS` 哪些值得进入真实实验
- 再基于现有 `PLY` 工具，补一轮针对底部离群高斯的定向参数扫描
- 最后再决定是否继续补更精细的 box 裁切，或转入遮罩方案设计
