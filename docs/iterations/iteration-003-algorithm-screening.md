# Iteration 003 Algorithm Screening

## 目标

为若水广场当前阶段筛选最值得优先验证的 `GS` 算法路线，避免默认把当前 `nerfstudio + splatfacto` 当成唯一主线。

## 当前问题画像

当前已经明确的问题不是单一的“质量不够好”，而是三类问题叠加：

- 几何与边缘问题：`floaters`、边缘拉花、底部大尺度离群高斯
- 场景规模问题：校园空中场景比常见物体级和小房间级实验更接近大场景
- 交付问题：当前 `rgb PLY` 仍约 `267 MiB`，即使传输压缩后也仍约 `200 MiB`

因此路线筛选不能只问“谁更清晰”，还要同时问：

- 谁更可能减少无效高斯或改善结构稳定性
- 谁更适合大场景训练与渲染
- 谁更接近 `Web MVP` 的交付约束

## 候选路线

### 1. `2DGS` / 几何优先路线

代表：

- `2D Gaussian Splatting`
- `Mip-Splatting`

当前价值判断：

- `2DGS` 的核心优势是几何更准，适合处理表面结构、伪影和网格质量问题
- `Mip-Splatting` 更偏向抗锯齿与伪影抑制，适合减少多尺度观察下的 aliasing 和闪烁
- 这条路线更像“让结果更干净、更像表面”，但不是天然的“大场景交付减重方案”

对本项目的意义：

- 如果当前问题的主因是校园边缘、地面和建筑表面被 3D 高斯拉花污染，这条线值得认真看
- 如果当前更大的痛点是大场景规模、加载体积和分层交付，这条线应排在结构化大场景方案之后

当前结论：

- 需要纳入主评估池
- 但暂时不应作为第一优先级的唯一主线

### 2. 大场景结构化 `GS` 路线

代表：

- `Scaffold-GS`
- `Octree-GS`
- `CityGaussian`
- `VastGaussian`

当前价值判断：

- 这类方法直接面向大场景训练、结构化表示、`LOD`、分块或分层渲染
- 对若水广场这种校园级空中场景，这条线与“体积、范围、可交付性”更直接相关
- 如果后续确实要从 `180` 张扩到 `300-600` 张甚至更高，这类方法比继续裸跑单体 `splatfacto` 更值得优先评估

对本项目的意义：

- `Scaffold-GS` 更适合关注减少冗余高斯、提高视角自适应表现
- `Octree-GS` 更适合关注 `LOD`、层级结构与一致实时渲染
- `CityGaussian` / `VastGaussian` 更适合关注 divide-and-conquer、分块训练和更大范围场景

当前结论：

- 这是当前最该前置评估的主线
- 至少要先判断其中是否存在一条能同时回应“规模 + 质量 + 交付”的路线

### 3. 压缩与轻量化交付路线

代表：

- `LightGaussian`

当前价值判断：

- 这条线主要回答“当前高斯结果怎么变得更轻、更适合发布”
- 它不能替代训练侧的大场景优化，但非常贴近当前 `Web MVP` 的真实痛点
- 如果能与当前训练输出衔接，就比重新换整条训练栈更务实

对本项目的意义：

- 非常适合当作“交付表示”和“训练表示”解耦的候选
- 适合在当前已有可看结果的前提下继续压缩体积

当前结论：

- 应与大场景结构化路线并列为高优先级
- 不该等到复训之后才看

## 当前优先级

### 第一优先级

- 大场景结构化 `GS`

原因：

- 在“允许重训”的前提下，当前更值得先解决的是校园级场景的训练组织、结构稳定性与多尺度渲染能力
- 交付压缩仍然重要，但更适合作为训练路线收敛后的下一层优化

### 第二优先级

- 压缩与轻量化交付
- `2DGS`
- `Mip-Splatting`

原因：

- `LightGaussian` 对交付体积很重要，但在接受重训后，不再需要压过训练主线
- `2DGS / Mip-Splatting` 很可能改善几何稳定性和伪影，但不直接保证更轻、更适合大场景交付

| `VastGaussian` | 大场景分块、并行训练、外观变化解耦 | 若后续扩量明显，这条线对训练成本与大范围一致性最有帮助 | 更偏“真正大场景”与并行优化；当前校园规模可能还没完全触发其优势 | 中等偏低 | `P2` | 先做纸面筛选，判断它是否过度面向城市场景 |
| `LightGaussian` | 压缩、蒸馏、量化，降低交付体积 | 与当前 `267 MiB -> 200 MiB` 的现实瓶颈最直接相关 | 可能依赖特定表示或额外压缩流程；未必改善几何质量 | 中等偏高，如果可接现有高斯表示会很有价值 | `P1` | 先确认其输入假设是否接近当前结果，再决定是接在现有导出后还是接在替代训练后 |

- `P1` 表示当前应优先进入下一轮纸面核查或最小实验设计
- `P2` 表示应纳入主评估池，但不该先于 `P1` 路线消耗主要时间

## 针对若水广场的筛选判断

### 最值得先看的主线

#### `Scaffold-GS / Octree-GS / CityGaussian`

原因：

- 当前问题已经不只是“局部脏高斯”，而是“校园级场景如何组织、训练、渲染、分层交付”
- 这类方法比原始单场景 `splatfacto` 更接近项目后续真正的扩量方向
- 即使最后不直接采用，也能帮助我们明确：后面该不该坚持单体场景训练

当前推荐顺序：

1. `Scaffold-GS`
2. `Octree-GS`
3. `CityGaussian`

原因：

- `Scaffold-GS` 最适合作为第一个重训入口，因为它最接近“自定义 `COLMAP` 场景可直接切换”的最小实验
- `Octree-GS` 紧随其后，因为它与 `LOD`、多尺度渲染和后续 `Web` 分层交付强相关
- `CityGaussian` 很强，但流程更重，更适合在确认要走分块训练后再投入

## 2026-03-24 基线实跑后补记

`Scaffold-GS` 已经完成首轮真实 baseline 验证，因此这里的优先级需要补一条约束：

- 它仍然是当前最容易接入、最先被验证通的结构化 `GS` 路线
- 但“接入优先级最高”不再等于“当前质量主线最优”

基于首轮结果：

- `Scaffold-GS baseline`：`PSNR 16.2589 / SSIM 0.3200 / LPIPS 0.5592`
- `Iteration 001 splatfacto`：`PSNR 20.25 / SSIM 0.599 / LPIPS 0.338`

当前判断应更新为：

- `Scaffold-GS` 仍保留在结构化大场景路线的首位，因为工程入口最清晰
- 但它在若水广场当前 `180` 张子集上的首轮 baseline 已经证明：默认参数下并没有优于现有 `splatfacto`
- 因此它现在应被视为“值得带假设继续验证的结构化备选路线”，而不是默认接管主线

### 不该忽略但暂排第二层的路线

#### `LightGaussian / 2DGS / Mip-Splatting`

原因：

- `LightGaussian` 仍然重要，但在接受重训后更适合作为训练路线收敛后的压缩层
- `2DGS / Mip-Splatting` 更像“质量和几何校正器”
- 对当前 `floaters`、边缘拉花和表面稳定性是有针对性的
- 但如果先把主力放到这条线上，容易绕开当前更紧的场景规模与训练组织问题

## 下一步的最小决策问题

在进入真实实验前，先回答这三个问题：

1. `Scaffold-GS` 是否应作为当前第一条真实重训主线？
2. `Octree-GS` 是否应作为紧随其后的 `LOD / Web` 导向备选？
3. `LightGaussian / 2DGS / Mip-Splatting` 是否都应该降为第二层优化分支？

## 建议下一步

1. 先围绕 `Scaffold-GS` 做最小重训入口核查
2. 再围绕 `Octree-GS` 核查其 `LOD` 路线是否值得紧接着跟进
3. 把 `LightGaussian` 改成训练路线收敛后的交付压缩备选
3. 再决定第一个真实实验是：
   - 继续基于当前 `PLY` 做定向清理
   - 迁移到压缩交付路线
   - 或为下一轮复训切到结构化大场景 `GS`
